1) Quick Start and Core Canvas Options

Instantiate a canvas in a DOM container, choose the renderer, and optionally apply theme defaults for node() and edge() helpers.

import Z from 'zeta.js';

const z = new Z.Canvas('#figure', {
  renderer: 'auto',     // 'canvas2d' | 'svg' | 'auto'
  width: 960,
  height: 540,
  responsive: true,
});

z.theme('diagram');

const a = z.node('API', { at: ['8.3%', '22.2%'], size: ['16%', '13%'] });
const b = z.node('Worker', { at: ['33.3%', '22.2%'], size: ['18%', '13%'] });
z.edge(a, b, { route: 'orthogonal' });

z.flush();
CanvasOptions: renderer, width, height, responsive
theme(nameOrTheme): built-ins include 'diagram' and 'slate'
Demo convention: component at/size uses %; keep px for low-level geometry when needed
getRenderer(), getScene(), flush(), loop(), dispose()

2) Primitives, Styling, Text, and LaTeX

Zeta primitives are chainable scene nodes. Use transform methods for placement and style methods for visual rules. Text nodes support both plain and LaTeX-normalized rendering.

import { Style } from 'zeta.js';

const card = Style.fill('rgba(56,189,248,0.14)').stroke('#38bdf8', 2);

z.rect([22, 22], [128, 80]).radius(14).useStyle(card);
z.circle([214, 64], 38).fill('#0b2a1e').stroke('#34d399', 2).dashed([7, 4]);

z.path([308, 18])
  .moveTo(20, 16)
  .lineTo(110, 2)
  .lineTo(170, 56)
  .lineTo(136, 128)
  .lineTo(44, 116)
  .close()
  .fill('rgba(244,114,182,0.16)')
  .stroke('#f472b6', 2);

z.line([168, 148], [490, 232])
  .route('step', { radius: 10 })
  .stroke('#fbbf24', 2)
  .dashed([8, 4]);

z.text('text() + font controls', [24, 156]).fontSize(12);
z.tex('\\frac{\\alpha + \\beta}{\\sqrt{x}}', [24, 194], { displayMode: true });

Live demo rendered with Canvas2D

Factories: rect(), circle(), path(), line(), text(), tex(), group()
Transforms: pos(), at(), move(), rotateTo(), scaleTo(), visible()
Styles: fill(), stroke(), dashed(), opacity(), cursor(), useStyle()

3) Anchors, Connectors, and Routing Modes

Use edge(from, to, opts) for reactive connectors. Choose route mode and obstacle handling. Access named anchors or perimeter ray sampling with anchor.atAngle().

const edge = z.edge(a, b, {
  from: 'right',
  to: 'left',
  route: 'orthogonal',
  routeOptions: {
    radius: 10,
    avoidObstacles: true,
  },
  color: '#38bdf8',
  width: 2,
  dash: [6, 4],
});

const boxTop = a.anchor.box.top;
const shapeRight = a.anchor.shape.right;
const sampled = a.anchor.atAngle(42);

z.line(a.anchor.center, sampled).stroke('#fbbf24', 1.6);

Straight, step, and orthogonal routes with live angle sampling.

Named anchors: top, bottom, left, right, center, corners
Anchor semantics: anchor.box.* (bbox) vs anchor.shape.* (perimeter)
Route options: straight, step, orthogonal, with radius and avoidObstacles

4) Constraints, Follow Bindings, and Interactivity

Relative layout stays live. Move one node and dependents follow automatically. Pointer events bubble, and drag helpers are built in.

const leader = z.node('Leader', { at: ['38.5%', '36%'], size: ['24.6%', '24%'] })
  .dragWithin([18, 26, 502, 286]);

const left = z.node('Left').follow(leader, 'left', { gap: 44, align: 'center' });
const right = z.node('Right').follow(leader, 'right', { gap: 44, align: 'center' });
const below = z.node('Below').follow(leader, 'below', { gap: 42, align: 'center' });

z.circle(6).follow(leader, 'topRight', { offset: [8, -8] });
z.edge(leader, right, { route: 'step', routeOptions: { radius: 10 } });

leader
  .hover(() => leader.opacity(1), () => leader.opacity(0.95))
  .click(() => leader.animate({ scale: 1.08 }, { duration: 120 }));

const unwatch = leader.watchLayout(() => {
  console.log('leader moved', leader.anchor.center);
});

Drag the leader; every follower and edge updates reactively.

Directional constraints: rightOf(), leftOf(), above(), below()
Binding helpers: pin(), follow(), watchLayout()
Events and drag: on(), off(), hover(), click(), draggable(), dragX(), dragY(), dragWithin()

5) Layout Macros, Panels, and Intent-Level Diagram APIs

Use macro containers to avoid coordinate bookkeeping. Add framed panels with container(), and build quick diagrams with node() + edge().

const panel = z.container({
  at: ['2.7%', '5%'],
  size: ['94.6%', '89.4%'],
  title: 'Workflow',
  fill: 'rgba(15,23,42,0.64)',
});

const content = panel.content;

const a = content.node('Ingest', { size: ['20.9%', '22.5%'] });
const b = content.node('Transform', { size: ['24.3%', '22.5%'] });
const c = content.node('Ship', { size: ['19.6%', '22.5%'] });

content.row([a, b, c], { gap: 14, align: 'center' }).at(['3.5%', '6.9%']);
content.edge(a, b, { from: 'right', to: 'left' });
content.edge(b, c, { from: 'right', to: 'left' });

content.column([...], { gap: 8, align: 'left' }).at(['3.5%', '45.9%']);
content.grid([...], { columns: 3, gap: [8, 8] }).at(['27.4%', '45.9%']);
content.stack([...], { align: 'topLeft', offset: [10, 10] }).at(['66.5%', '47.6%']);

Row, column, grid, and stack macros inside a titled container.

Macros: row(), column(), grid(), stack() on both canvas and groups
Panel helper: container(opts) exposes frame, titleNode, and content
Labeled nodes: node(label, opts) supports subtitle and ports

6) Context Groups: Data Coordinates and Isometric Projection

Group context changes coordinate behavior. Use coords() for data space and project('isometric') for pseudo-3D. 

const plot = z.group()
  .at(['3.5%', '8.7%'])
  .size(['44.6%', '62.7%'])
  .coords({
    x: { domain: [0, Math.PI * 2] },
    y: { domain: [-1.2, 1.2] },
  });

plot.axes({ grid: true, xLabel: 'x', yLabel: 'f(x)', tickCount: 5 });
plot.func((x) => Math.sin(x), { samples: 120 }).stroke('#38bdf8', 2);
plot.func((x) => 0.5 * Math.cos(2 * x), { samples: 120 }).stroke('#f472b6', 2);
plot.circle([Math.PI / 2, 1], 4);

const iso = z.group().at(['67.7%', '64.7%']).project('isometric', { angle: 30, scale: 15 });
iso.line([0, 0, 0], [6, 0, 0]);
iso.line([0, 0, 0], [0, 6, 0]);
iso.line([1, 1, 0], [1, 1, 1.2]);

Left: plotting coordinates. Right: isometric projection with 3D tuples.

Plotting APIs: size(), coords(), axes(), func()
Projection API: project('isometric', { angle, scale })
Group-local factories accept tuples in context space: group.rect/circle/text/path/line

const stop = z.loop((time, dt) => {
  const t = (Math.sin(time * 0.002) + 1) / 2;

8) Themes and Geometry Debug Snapshots

Apply default visual recipes with theme() and inspect resolved geometry with debugSnapshot(). The snapshot gives bounds, anchors, and route points for custom overlays.

z.theme('diagram');

const a = z.node('A', { at: ['8.1%', '26.9%'] });
const b = z.node('B', { at: ['41.9%', '56.6%'] });
const c = z.node('C', { at: ['75%', '29%'] });

const ab = z.edge(a, b, {
  route: 'orthogonal',
  routeOptions: { radius: 10, avoidObstacles: true },
});

const snapshot = z.debugSnapshot({
  bounds: true,
  anchors: true,
  routes: true,
});

// draw your own overlays from snapshot.bounds / snapshot.anchors / snapshot.routes

Drag nodes to see the debug overlay (bounds, route traces, sampled anchors) refresh.

theme(nameOrTheme) merges defaults rather than replacing everything
debugSnapshot(opts) returns structured diagnostics for custom dev tooling

9) Plugin API: Extend Canvas and Group Methods

Plugins run once via Z.use(). defineShape() and defineMacro() register methods on both Canvas and Group hosts.

import Z from 'zeta.js';

type Unit = number | `${number}%`;

type PillAttrs = {
  at?: [Unit, Unit];
  size?: [Unit, Unit];
  fill?: string;
};

Z.defineShape<PillAttrs>('pill', (host, attrs = {}) => {
  const at = attrs.at ?? [0, 0];
  const size = attrs.size ?? [120, 40];
  const fill = attrs.fill ?? 'rgba(56,189,248,0.15)';

  return host.rect(at, size)
    .radius(size[1] / 2)
    .fill(fill)
    .stroke('#38bdf8', 1.6);
});

Z.defineMacro('pair', (host) => {
  const left = host.node('Left', { at: ['8%', '10%'] });
  const right = host.node('Right', { at: ['42%', '10%'] });
  host.edge(left, right, { route: 'straight' });
  return { left, right };
});

const z = new Z.Canvas('#figure');
const pill = z.pill({ at: ['12%', '40%'] });
const pair = z.pair();
Z.use(plugin): idempotent plugin registration
Z.defineShape(name, factory) and Z.defineMacro(name, factory) reject duplicate names
Z.midpoint(a, b) helper returns tuple midpoint from world bbox centers

10) Reactive Utilities and Math Helpers

Zeta exports lightweight reactive primitives and geometry classes for advanced behaviors and tooling.

import {
  signal,
  computed,
  batch,
  Vec2,
  BBox,
  Matrix3,
  perimeterPoint,
  rayShapeIntersection,
} from 'zeta.js';

const zoom = signal(1);
const accent = signal('#38bdf8');

const strokeWidth = computed(() => Math.max(1, zoom.get() * 1.4));
strokeWidth.subscribe((w) => console.log('new width', w));

batch(() => {
  zoom.set(1.5);
  accent.set('#34d399');
});

const center = new Vec2(100, 100);
const bbox = BBox.fromPosSize(60, 60, 80, 50);
const hit = perimeterPoint(center, 45, { type: 'rect', bbox });
const world = Matrix3.translate(20, 10).transformPoint(hit);
Reactive exports: Signal, Computed, signal(), computed(), batch()
Math exports: Vec2, BBox, Matrix3, intersection helpers

11) Renderer Access, SVG Export, and Cleanup

Renderer access is useful for diagnostics and exports. If using SVG renderer, you can serialize the scene. Always dispose when a canvas is no longer needed.

import Z, { SVGRenderer } from 'zeta.js';

const z = new Z.Canvas('#figure', { renderer: 'svg', width: 800, height: 420 });

// build scene...
z.flush();

const renderer = z.getRenderer();
if (renderer instanceof SVGRenderer) {
  const svgText = renderer.export();
  console.log(svgText);
}

const stop = z.loop((time, dt) => {
  // update scene over time
});

// later:
stop();
z.dispose();
flush() forces synchronous rendering, useful for tests and snapshot workflows
dispose() detaches listeners, clears scene internals, and removes renderer DOM
